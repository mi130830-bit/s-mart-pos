คู่มือการใช้งานระบบ POS & S-Link

ระบบบริหารจัดการหน้าร้านแบบ Hybrid (Offline/Online) พร้อมระบบจัดการงานขนส่ง

1. ภาพรวมระบบ

ระบบ S-Link POS ถูกออกแบบมาเพื่อให้ร้านค้าสามารถดำเนินงานขายได้รวดเร็วแม้ไม่มีอินเทอร์เน็ต (Local MySQL) และเชื่อมต่อข้อมูลลูกค้าและงานขนส่งผ่าน Cloud (Firebase) เพื่อให้ทีมส่งของทำงานได้ทันที

🖥️
POS Desktop

โปรแกรมหลักบน Windows สำหรับแคชเชียร์ พิมพ์บิล จัดการสต็อก และดูรายงาน

📱
S-Link Mobile

แอปพลิเคชันบนมือถือสำหรับดูสต็อก ยิงบาร์โค้ด และจัดการงานส่งของ (Delivery Job)

🖨️
Hybrid Printing

รองรับทั้งเครื่องพิมพ์สลิปความร้อน (80mm) และเครื่องพิมพ์เอกสาร (A4/A5 Dot Matrix)

2. การตั้งค่าเริ่มต้น (Initial Setup)

เมื่อติดตั้งโปรแกรมครั้งแรก ต้องทำการตั้งค่าเพื่อให้ระบบเชื่อมต่อกับฐานข้อมูลและอุปกรณ์ต่อพ่วงได้ถูกต้อง ไปที่เมนู Settings (ตั้งค่า)

2.1 การตั้งค่าฐานข้อมูล (Database Config)

Local Database (MySQL) ระบุ Host (เช่น 127.0.0.1 หรือ IP ของเครื่อง Server), Port (3306), Username และ Password
Cloud Database (Firestore) ระบบจะเชื่อมต่ออัตโนมัติหากมีไฟล์ Configuration (google-services.json) อยู่ในโฟลเดอร์ที่กำหนด

2.2 ตั้งค่าข้อมูลร้าน (Shop Info)

กรอกชื่อร้าน, ที่อยู่, เบอร์โทร และ เลขประจำตัวผู้เสียภาษี ข้อมูลเหล่านี้จะปรากฏบนหัวใบเสร็จและใบกำกับภาษี

2.3 ตั้งค่าเครื่องพิมพ์ (Printer Settings) Update

ประเภทการพิมพ์ รายละเอียด
Short Receipt (POS Slip) สำหรับสลิปย่อ 80mm/58mm (Thermal) ใช้สำหรับลูกค้าทั่วไป
Cash Bill (บิลเงินสด Full) สำหรับบิลเงินสดเต็มรูปแบบ A4/A5 (Dot Matrix/Inkjet) ตั้งค่าแยกจากสลิปได้
Delivery Note ใบส่งของ (A4/A5) สำหรับงานขนส่ง
Label/Barcode สำหรับพิมพ์สติ๊กเกอร์บาร์โค้ด

2.4 จอแสดงผลลูกค้า (Customer Display)

หากเครื่อง POS มี 2 หน้าจอ ให้เปิดฟังก์ชัน Enable Second Screen ระบบจะแสดงราคารวมและ QR Code พร้อมเพย์ที่หน้าจอที่ 2 ให้ลูกค้าเห็นอัตโนมัติ

3. ระบบขายหน้าร้าน (POS)

หน้าจอหลักสำหรับการขาย ออกแบบมาให้ทำงานเร็วด้วยคีย์ลัดและการยิงบาร์โค้ด

🛍️ ขั้นตอนการขาย
  1. ยิงบาร์โค้ดสินค้า หรือพิมพ์ค้นหาชื่อ
  2. ระบุจำนวน (ถ้ามากกว่า 1)
  3. กด Checkout (F1)
💳 การชำระเงิน
  • Cash: ใส่ยอดเงินรับ ระบบคำนวณเงินทอน
  • Transfer: จอจะแสดง QR Code รับเงิน
  • Credit: บันทึกเป็นลูกหนี้ (ต้องเลือกลูกค้าก่อน)
🔄 การพิมพ์ซ้ำ (Reprint)

ในหน้า Dashboard สามารถกด Reprint และเลือกรูปแบบได้หลากลาย:

  • สลิป 80mm / 58mm
  • บิลเงินสด A4 / A5
  • ใบส่งของ A4 / A5

4. การจัดการสต็อก (Inventory)

จัดการข้อมูลหลัก (Master Data) New

  • Product Types: กำหนดประเภทสินค้าแยกย่อย (เช่น ฮาร์ดแวร์, เคมีภัณฑ์) พร้อมตั้งค่าว่าเป็นสินค้าชั่งน้ำหนักหรือไม่
  • Units: จัดการหน่วยนับสินค้า (ชิ้น, กล่อง, kg, เมตร)

เมนูสินค้า (Products)

ใช้สำหรับเพิ่มสินค้าใหม่ แก้ไขราคา หรือพิมพ์บาร์โค้ด

  • Add New: เพิ่มสินค้า ใส่ชื่อ, บาร์โค้ด, ราคาทุน, ราคาขาย
  • Barcode Designer: ออกแบบและพิมพ์สติ๊กเกอร์บาร์โค้ด เลือกขนาดกระดาษได้

ประวัติสินค้าและการสั่งซื้อ (Product History) New

เมื่อกดดูรายละเอียดสินค้า จะมี Tab ข้อมูลเพิ่มเติม:

  • ประวัติเคลื่อนไหว (Movement): ดูรายการ เข้า/ออก ของสต็อก
  • ข้อมูลสั่งซื้อ (Restock Info): ดูประวัติการซื้อย้อนหลัง ระบบจะไฮไลท์ ราคาดีที่สุด (Best Price) ให้ทราบทันที เพื่อช่วยตัดสินใจสั่งของเข้า

การปรับสต็อก

  • Stock In/Out: รับสินค้าเข้าหรือเบิกออกเพื่อใช้งาน
  • Stock Check: ตรวจนับสต็อกจริงเปรียบเทียบกับในระบบ

5. ลูกค้าและลูกหนี้ (CRM & Debtor)

การจัดการลูกค้า

บันทึกชื่อ, เบอร์โทร, ที่อยู่ และพิกัด GPS (สำหรับจัดส่ง)

ระบบลูกหนี้ (Credit/Debtor) Desktop

เมื่อมีการขายเชื่อ (Credit Sale):

1. บันทึกขายเชื่อ: ในหน้า POS เลือกชำระเงินแบบ "Credit" ระบบจะบังคับให้เลือกลูกค้า
2. ติดตามหนี้ (Debtor List): ดูรายการที่ยังค้างชำระ ยอดรวมหนี้สิน และประวัติการผ่อนชำระ
3. รับชำระหนี้: สามารถรับชำระบางส่วน หรือปิดยอดทั้งหมดได้

7. การแก้ปัญหาเบื้องต้น

  • เชื่อมต่อ Database ไม่ได้: ตรวจสอบว่าเครื่อง Server เปิดอยู่ และสาย LAN เชื่อมต่อปกติ หรือตรวจสอบ IP Address ในหน้า Settings
  • พิมพ์ไม่ออก: ตรวจสอบสาย USB เครื่องพิมพ์ หรือดูว่าDriver เครื่องพิมพ์สถานะเป็น "Ready" หรือไม่
  • ภาษาต่างดาวบนสลิป: ตรวจสอบการตั้งค่า Codepage ใน Printer Settings (ควรเป็น Thai/Vietnamese หรือ UTF-8)
